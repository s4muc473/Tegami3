8 IPS: 6 COMPUTADORES

Certo! Se você precisa criar uma rede com 8 IPs, você precisa usar uma máscara de sub-rede que permita pelo menos 8 endereços IP, e vou te mostrar como fazer isso de maneira detalhada.

Para criar uma rede com 8 endereços IP utilizáveis, você precisa escolher uma máscara de sub-rede /29. Vamos entender tudo direitinho!

Máscara de Sub-rede /29 (255.255.255.248)
A máscara /29 vai criar uma rede com 8 endereços no total, sendo que 6 desses endereços são válidos para hosts (dispositivos), e os outros dois são reservados:

1 endereço para a rede
1 endereço para o broadcast
Com a máscara /29, a estrutura fica da seguinte forma:

Total de endereços: 8
Endereços válidos para hosts: 6
Endereço da rede: 1 (não é atribuído a um dispositivo, é apenas para identificar a rede)
Endereço de broadcast: 1 (também não é atribuído a um dispositivo)
Cálculo do intervalo de IPs com a máscara /29
Quando usamos a máscara 255.255.255.248 (/29), ela cria um intervalo de 8 endereços por bloco de rede. Esse bloco vai do primeiro endereço da sub-rede até o último, incluindo a rede e o broadcast. O intervalo será o seguinte:

Primeiro endereço: Identifica a rede (não pode ser atribuído a um dispositivo).
Último endereço: É o endereço de broadcast (também não pode ser atribuído a um dispositivo).
Os endereços entre o primeiro e o último são válidos para atribuição aos dispositivos da rede.
Vamos pegar um exemplo prático.

Exemplo: Criando uma rede de 8 IPs com máscara /29
Passo 1: Definir o endereço da rede
Vamos supor que você tenha o endereço 192.168.1.0. Agora, vamos usar a máscara /29 para dividir essa rede em blocos menores.

Rede: 192.168.1.0/29
Passo 2: Determinar os IPs utilizáveis
Com a máscara /29, os IPs válidos serão os seguintes:

Rede: 192.168.1.0 (Este é o endereço da rede. Não pode ser atribuído a nenhum dispositivo).
IP 1: 192.168.1.1 (Este é o primeiro IP válido. Pode ser atribuído a um dispositivo, como o PC1).
IP 2: 192.168.1.2 (Este é o segundo IP válido. Pode ser atribuído a outro dispositivo, como o PC2).
IP 3: 192.168.1.3 (Outro dispositivo válido).
IP 4: 192.168.1.4 (Outro dispositivo válido).
IP 5: 192.168.1.5 (Outro dispositivo válido).
IP 6: 192.168.1.6 (Outro dispositivo válido).
Broadcast: 192.168.1.7 (Este é o endereço de broadcast. Não pode ser atribuído a nenhum dispositivo).
Passo 3: Atribuindo os endereços aos dispositivos
Agora, você pode atribuir os IPs válidos aos dispositivos da rede, por exemplo:

PC1: IP 192.168.1.1, Máscara 255.255.255.248
PC2: IP 192.168.1.2, Máscara 255.255.255.248
Switch/Outro Dispositivo: IP 192.168.1.3, Máscara 255.255.255.248
Servidor: IP 192.168.1.4, Máscara 255.255.255.248
Outro Dispositivo: IP 192.168.1.5, Máscara 255.255.255.248
Outro Dispositivo: IP 192.168.1.6, Máscara 255.255.255.248
Passo 4: Verificando o Endereço de Broadcast
O endereço de broadcast para essa rede será 192.168.1.7. Este é o endereço usado para enviar pacotes para todos os dispositivos da rede ao mesmo tempo (broadcast). Você não pode atribuir esse endereço a nenhum dispositivo.

Esquema completo de rede:
Dispositivo	IP	Máscara de Sub-rede	Observações
Rede (identificador)	192.168.1.0	255.255.255.248	Não pode ser usado
PC1	192.168.1.1	255.255.255.248	
PC2	192.168.1.2	255.255.255.248	
Outro Dispositivo	192.168.1.3	255.255.255.248	
Servidor	192.168.1.4	255.255.255.248	
Outro Dispositivo	192.168.1.5	255.255.255.248	
Outro Dispositivo	192.168.1.6	255.255.255.248	
Endereço de Broadcast	192.168.1.7	255.255.255.248	Não pode ser usado
Resumo:
Endereço de rede: 192.168.1.0
Máscara de sub-rede: 255.255.255.248 (/29)
IPs válidos para hosts: 192.168.1.1 a 192.168.1.6
Endereço de broadcast: 192.168.1.7
Quantidade de endereços válidos: 6
Essa configuração vai criar uma rede com 8 IPs totais (1 para a rede, 1 para o broadcast, e 6 válidos para os dispositivos).

Passos no Packet Tracer:
Abra o Cisco Packet Tracer.
Arraste dois PCs (ou mais, conforme sua necessidade) para a tela.
Conecte os PCs a um switch ou roteador (se for o caso).
Clique nos PCs, entre em "Configuração" e defina os IPs e a máscara:
PC1: IP 192.168.1.1, Máscara 255.255.255.248
PC2: IP 192.168.1.2, Máscara 255.255.255.248
E assim por diante para os outros dispositivos.
Teste a conectividade entre os dispositivos com o comando ping.
Agora sua rede está pronta para ser usada! Qualquer dúvida ou se precisar de mais detalhes, só me avisar!

Máscara /30 (255.255.255.252)
A máscara /30 divide a rede em blocos de 4 endereços IP, onde:

1 endereço é reservado para a rede (não pode ser atribuído a um dispositivo),
1 endereço é reservado para o broadcast (também não pode ser atribuído a um dispositivo),
2 endereços IP válidos para hosts (que são atribuídos aos dispositivos da rede).
Portanto, com uma máscara /30, você tem:

Total de IPs: 4
IPs válidos para hosts: 2
Endereço da rede: 1
Endereço de broadcast: 1
Como configurar uma rede com 2 IPs
Vamos agora criar uma rede usando a máscara /30.

Exemplo: Rede com 2 IPs válidos (ponto a ponto)
Suponha que você tenha o endereço 192.168.1.0/30. Essa rede irá fornecer os seguintes endereços:

Rede: 192.168.1.0 (não pode ser atribuído a um dispositivo, é a identificação da rede).
IP 1 (válido para host): 192.168.1.1 (pode ser atribuído a um dispositivo, como um roteador ou PC).
IP 2 (válido para host): 192.168.1.2 (pode ser atribuído a outro dispositivo, como outro roteador ou PC).
Broadcast: 192.168.1.3 (não pode ser atribuído a um dispositivo, é usado para enviar pacotes de broadcast para a rede).
Passo a Passo de Configuração
Endereço de Rede: 192.168.1.0/30
IP válido para host 1: 192.168.1.1
IP válido para host 2: 192.168.1.2
Endereço de Broadcast: 192.168.1.3
Então, para atribuir os endereços IP aos dispositivos, você faria o seguinte:

Roteador 1 (ou o primeiro dispositivo): Atribua o IP 192.168.1.1 e a máscara 255.255.255.252.
Roteador 2 (ou o segundo dispositivo): Atribua o IP 192.168.1.2 e a máscara 255.255.255.252.
Lembre-se: O IP 192.168.1.0 é reservado para a rede, e 192.168.1.3 é o endereço de broadcast. Nenhum desses pode ser atribuído a um dispositivo.
